## Claude Code Specifics

- The detailed brain (`tokens.md`, `components/`, `motion.md`, etc.) is not auto-loaded.
  Read the relevant file before acting; the routing table in `AGENTS.md` tells you which.
- For repeatable design work, prefer the skills in `design-brain/skills/`:
  - `component-contract` — build/refactor a component to its written contract.
  - `extract-contract` — generate a contract from existing source.
  - `port-to-orbit` — bring an external/Lovable prototype onto Orbit tokens/components.
- For multi-component or full-screen work, delegate review to `design-reviewer`
  (`design-brain/agents/design-reviewer.md`). A blocker means not done.

## Build / Test Commands

Run from `/Users/derekwong/efficio-orbit` unless stated otherwise.

- Install: `npm install`
- Dev docs: `npm run dev:docs`
- Dev prototypes: `npm run dev:prototypes`
- Build all workspaces: `npm run build`
- Build component package: `npm run build:orbit`
- Typecheck component package: `npm run typecheck:orbit`
- Test components: `npm run test:components`
- Lint: `npm run lint`
- Static design-system audit: `npm run audit:design-system`
- Benchmark accessibility artifact: `npm run bench:a11y`
- Browser visual accessibility pass: see
  `design-brain/_benchmarks/accessibility-artifact-process.md`
- Regenerate generated design docs: `npm run build:design-md`
- Export Orbit style guide: `npm run export:style-guide`
