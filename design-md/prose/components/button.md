Use Button for the actions on a view. Reserve the `Primary` variant for the single most important action in a region — never show two primary buttons side by side. `Secondary` is the default for most actions, `Tertiary` for low-emphasis or repeated ones, and `Positive`/`Destructive` only when the outcome is genuinely additive or irreversible.

- Do lead each view with one clear primary action.
- Do use `Small` only in dense toolbars and table rows.
- Don't use `Destructive` for routine actions — it is reserved for delete/remove.
- Don't pair an icon with a label that just repeats it; the icon should add meaning.
