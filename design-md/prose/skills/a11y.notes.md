## Contrast

Orbit targets **WCAG 2.2 AA**:

- **4.5:1** for body text against its background.
- **3:1** for large text (≥ 24px, or ≥ 18.66px bold) and for non-text UI (borders,
  icons, control boundaries, focus rings).

Never communicate state with colour alone. Error states pair the error colour with
an icon **and** accessible text; status chips carry a label, not just a hue.

## Keyboard & focus

- Every interactive element is reachable and operable by keyboard. Don't replace a
  native control with a styled `div`.
- Focus is always visible — the focus ring uses `--orbit-color-focus-ring`. Don't
  remove outlines without providing an equivalent visible indicator.
- Overlays and dialogs (`Overlay`) expose a dialog name, keep focus inside while
  open, and **restore focus** to the trigger on close.
- Containers stay non-interactive. `Card` is a container, not a button — put the
  action on a child button or link rather than a `role="button"` wrapper.

## Naming

Inputs require a deliberate accessible name (visible `label`, `ariaLabel`, or
`ariaLabelledBy`). See `references/input-naming-contract.md` for the per-component
contract; the dev-only `warnIfUnnamed` helper flags any field left unnamed.
