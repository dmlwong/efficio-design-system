# Golden Example: Orbit Client Home AI Tools Dashboard

## Demonstrates
- Client Connected Platform home/dashboard treatment.
- Dark persistent navigation with client-facing work surface.
- AI tools and performance summary hierarchy.

## Pattern Represented
Client-facing home dashboard for Orbit / Client Connected Platform.

Use this when creating or reviewing client home, dashboard, or tool-entry surfaces.

## Components Visible
- Persistent shell/navigation.
- Tool cards.
- Dashboard summary cards.
- Initiative/performance summary regions.
- Status and count indicators.

## Layout Guidance
- Keep the client shell persistent and clearly separated from the work surface.
- The page should foreground useful tools and current work, not marketing copy.
- Use cards to guide the client toward concrete products, summaries, and next actions.
- Keep dashboard regions legible and grouped around client outcomes.

## Density Guidance
- Preserve enterprise density, but allow more guidance and grouping than internal CP.
- Avoid sparse landing-page spacing.
- Keep card grids aligned and action-oriented.

## Copy Guidance
- Use client-safe labels and avoid internal Efficio shorthand.
- Explain tool purpose and dashboard state in plain language.
- Avoid vague AI hype; connect AI tooling to procurement tasks and evidence.

## Imitate
- Client shell rhythm.
- Tool-first dashboard hierarchy.
- Clear work summaries.
- Client-safe action labels.

## Do Not Imitate
- Generic SaaS homepage hero.
- Internal CP operational shortcuts.
- Decorative feature cards with no real workflow path.
- Benchmark/docs screenshots as visual truth.

## Review Questions
- Which AI tool labels are approved for client-facing use?
- Which dashboard summaries need source-backed component contracts?
- Which empty/loading dashboard states should be captured next?
