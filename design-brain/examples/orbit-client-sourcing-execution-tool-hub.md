# Golden Example: Orbit Client Sourcing Execution Tool Hub

## Demonstrates
- Client-facing tool hub for sourcing execution.
- Product cards for MarketIQ, RFP Builder, RFP Analytics, and ClauseIQ.
- Guided entry into procurement workflows.

## Pattern Represented
Tool hub / product selection page for Orbit / Client Connected Platform.

Use this when creating or reviewing pages where external users choose among major
workflow tools.

## Components Visible
- Tool/product cards.
- Shell/navigation.
- Product area header.
- Primary tool-entry actions.
- Short descriptions and status/context metadata.

## Layout Guidance
- Cards should represent real tools, not generic feature marketing.
- Each tool card needs a clear purpose and entry action.
- Keep the hub scannable; users should quickly compare tools and know where to start.
- Preserve the current shell and product-area context.

## Density Guidance
- Tool cards can be more spacious than CP operational cards, but should not become
  oversized promotional tiles.
- Keep descriptions short and useful.
- Ensure the card grid works at both comfortable and compact enterprise rhythms.

## Copy Guidance
- Use client-safe product names and descriptions.
- Explain what each tool helps the user do.
- Avoid internal abbreviations unless they are visible in the client product source.

## Imitate
- Real tool names.
- Concrete tool-entry actions.
- Client-safe summaries.
- Clear card comparison.

## Do Not Imitate
- Generic feature cards detached from real products.
- Internal-only process language.
- Marketing-page hero composition.
- Duplicate CTAs that obscure the primary tool choice.

## Review Questions
- Are the current tool descriptions approved client-facing copy?
- Which tool availability states need screenshots?
- Should unavailable or coming-soon tools use a dedicated status contract?
