# Orbit tokens

Orbit is Efficio's design system for AI-orchestrated procurement intelligence. This
skill is the authoritative source for Orbit's **design tokens** — the exact colour,
spacing, typography, radius, and elevation values, and the rules for referencing them.

## How to reference tokens

- Tokens are **CSS custom properties** prefixed `--orbit-`. Reference them with
  `var(--orbit-…)` (or, in `DESIGN.md` YAML, `{colors.token}`).
- **Never hard-code a raw value** (hex, px, etc.) in a reusable component — the
  static audit (`npm run audit:design-system`) fails on raw hex in
  `packages/orbit/src`. Reach for the semantic token first, then the primitive.
- Prefer **semantic** colour tokens (`text-*`, `border-*`, `bg-*`) over primitives
  for anything that conveys meaning; primitives are the underlying palette.

## Theme

Orbit is **light-only — there is no dark mode.** The base `:root` palette ("CP
mode") is **canonical**. A single brand variant, **Orbit mode**, is activated by
`data-theme="orbit"` on an ancestor and re-skins some colours; it is a re-skin, not
a light/dark toggle. Its overrides are documented in `references/theme-orbit.md`.
